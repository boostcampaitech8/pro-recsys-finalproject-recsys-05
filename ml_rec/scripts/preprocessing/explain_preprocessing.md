## 🎯 왜 이렇게 필터링했는가?

### **문제 상황**
```
원본 데이터로 EASE 학습 시도
→ Killed (메모리 부족, 시간 초과)
```

### **해결 전략**
```
1. 메모리 극적 감소 필요
   → K30 + 활동 범위 필터링으로 14.7배 감소

2. 데이터 품질 우선
   → 고립된 노드 제거 (K-core)
   → 극단값 제거 (User/Item Range)

3. 추천 시스템에 최적화
   → 활발한 사용자 + 인기있는 게임만 유지
   → 명확한 추천 신호만 보존
```

## 💡 각 필터링 기준의 의미

### **K-core (k=30)**
- **그래프 이론 기반**: 모든 노드가 최소 30개 연결 보유
- **의미**: 고립되지 않은 노드만 유지
- **효과**: 매우 견고하고 연결된 그래프 구조
- **사용 모델**: LightGCN, GNN 기반 모델에 이상적

### **User Activity Range (20-500)**
- **최소 20**: 너무 활동이 적은 사용자 제거
- **최대 500**: 비정상적 패턴(봇, 대량구매) 제거
- **의미**: 일반적인 플레이 패턴 사용자만 포함
- **효과**: 신뢰할 수 있는 선호도 신호

### **Item Popularity Range (20-1000)**
- **최소 20**: 너무 마이너 게임 제거
- **최대 1000**: 너무 인기있는 게임 제외
- **의미**: 중간 정도 인기도의 게임 집중
- **효과**: 명확한 차별성이 있는 게임만 유지
