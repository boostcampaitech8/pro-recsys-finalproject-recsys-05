# 📚 모델 가이드

Steam 게임 추천 시스템에 사용된 세 가지 추천 모델(EASE, BPR, LightGCN)에 대한 상세 설명입니다.

---

## 1. EASE (Embarrassingly Shallow Autoencoders)

### 📖 모델 개요

EASE는 매우 간단하면서도 강력한 선형 추천 모델입니다. 각 아이템에 대한 가중치를 학습하여 사용자-아이템 상호작용을 재구성합니다.

**특징:**
- **모델 타입**: 전통적 선형 모델
- **학습 파라미터**: 1개 (매우 적음)
- **연산 복잡도**: O(n²) - 간단
- **설명 가능성**: 높음 (명확한 가중치)

### 🔬 알고리즘 원리

사용자-아이템 상호작용 행렬 X를 선형 자동인코더로 간주하고, 다음 최적화 문제를 풀이합니다:

```
minimize: ||X - XB||²_F + λ||B||²_2
subject to: diag(B) = 0  (자기 상관 제거)
```

**의미:**
- B: 각 아이템의 추천 가중치 벡터
- λ: 정규화 강도
- diag(B) = 0: 아이템 자신으로는 학습 안 함

### 📊 성능 (steam_filtered_k30_activity)

**데이터셋:**
```
사용자: 29,153명
게임: 13,078개
상호작용: 1,969,577개
희소성: 99.48%
```

**학습 설정:**
```
에포크: 1
배치 크기: 2,048
옵티마이저: Adam (학습률 0.001)
정규화 계수: 250.0
```

**최고 성능:**

| 메트릭 | 검증 | 테스트 |
|--------|------|--------|
| **NDCG@10** | **0.4652** ⭐ | 0.4729 |
| Recall@10 | 0.4504 | 0.4546 |
| Recall@20 | 0.5632 | 0.5672 |
| NDCG@20 | 0.5044 | 0.511 |
| MRR@10 | 0.6968 | 0.7059 |
| MRR@20 | 0.6995 | 0.7085 |

**학습 통계:**
- 에포크 당 학습 시간: 5.35초
- 에포크 당 평가 시간: 3317초 (약 55분)
- GPU 메모리: 0.02G / 31.74G
- 총 메모리: 1.96G / 88.39G

### ✅ 장점 / ❌ 단점

| 장점 | 단점 |
|------|------|
| ✅ 매우 빠른 학습 | ❌ 새로운 아이템(Cold Start) 어려움 |
| ✅ 높은 성능 | ❌ 선형 관계만 학습 |
| ✅ 적은 파라미터 | ❌ 복잡한 패턴 표현 제한 |
| ✅ 해석 가능 | - |

### 🎯 적용 사례

- **추천 적합도**: ⭐⭐⭐⭐⭐ (최고)
- **포트폴리오 추천도**: ⭐⭐⭐⭐⭐ (가장 좋음)
- **프로덕션**: ⭐⭐⭐⭐ (배포 용이)

## 2. BPR (Bayesian Personalized Ranking)

### 📖 모델 개요

BPR은 쌍별 순위 학습(Pairwise Ranking Learning)을 기반한 신경망 기반 추천 모델입니다. 사용자가 상호작용한 아이템을 상호작용하지 않은 아이템보다 선호한다는 가정 하에 학습합니다.

**특징:**
- **모델 타입**: 신경망 기반 (일반)
- **학습 파라미터**: ~5.5M개
- **입력 방식**: Pairwise (긍정-부정 쌍)
- **손실 함수**: BPR Loss

### 🔬 알고리즘 원리

다음 최적화 목표를 가집니다:

```
maximize: Σ ln(σ(ŷ_ui - ŷ_uj))

여기서:
- σ: sigmoid 함수
- ŷ_ui: 사용자 u가 아이템 i를 선호할 확률
- ŷ_uj: 사용자 u가 아이템 j를 선호할 확률
- i: 상호작용한 아이템, j: 부정 샘플
```

**의미:**
- 상호작용한 아이템의 점수가 상호작용하지 않은 아이템보다 높아야 함
- 쌍별 순위 학습으로 순위 관계 학습

### 📊 성능 (steam 전체 데이터)

**데이터셋:**
```
사용자: 40,001명
게임: 45,756개
상호작용: 5,593,836개
희소성: 99.69%
```

**학습 설정:**
```
에포크: 50
배치 크기: 2,048
옵티마이저: Adam (학습률 0.001)
임베딩 크기: 64
부정 샘플: Full 전략
```

**최고 성능 (에포크 42):**

| 메트릭 | 검증 | 테스트 |
|--------|------|--------|
| **NDCG@10** | **0.1812** | 0.2059 |
| Recall@10 | 0.166 | 0.1776 |
| Recall@20 | 0.2342 | 0.2465 |
| NDCG@20 | 0.1959 | 0.2176 |
| MRR@10 | 0.3163 | 0.3572 |

**학습 통계:**
- 에포크 당 학습 시간: 13.5초
- 에포크 당 평가 시간: 47.5초
- 조기 종료: 에포크 42에서 달성
- GPU 메모리: 0.20G / 31.74G

### ✅ 장점 / ❌ 단점

| 장점 | 단점 |
|------|------|
| ✅ 신경망 기반 (유연함) | ❌ EASE보다 낮은 성능 |
| ✅ 순위 기반 학습 | ❌ 더 많은 파라미터 |
| ✅ 임베딩 활용 | ❌ 더 느린 학습 |
| ✅ 다양한 변형 가능 | - |

### 🎯 적용 사례

- **추천 적합도**: ⭐⭐⭐ (중간)
- **포트폴리오 추천도**: ⭐⭐⭐⭐ (비교 분석 용도)

## 3. LightGCN (Light Graph Convolutional Network)

### 📖 모델 개요

LightGCN은 그래프 신경망(GNN) 기반의 현대적 추천 모델입니다. 사용자-아이템 이분 그래프에서 그래프 컨볼루션을 통해 사용자와 아이템의 임베딩을 학습합니다.

**특징:**
- **모델 타입**: 그래프 신경망 (GNN)
- **학습 파라미터**: ~17.4M개
- **그래프 레이어**: 2층
- **입력 방식**: Pairwise (긍정-부정 쌍)

### 🔬 알고리즘 원리

사용자-아이템 이분 그래프를 구성하고 그래프 컨볼루션을 적용합니다:

```
그래프 구조:
User1 ─┬─ Item1
       ├─ Item2
User2 ─┼─ Item3
       └─ Item4

그래프 컨볼루션:
e_u^(l+1) = σ(Σ_{i∈N_u} 1/√|N_u||N_i| e_i^(l))
e_i^(l+1) = σ(Σ_{u∈N_i} 1/√|N_u||N_i| e_u^(l))
```

**의미:**
- 이웃 노드들의 임베딩을 이용해 자신의 임베딩 업데이트
- 그래프 구조를 명시적으로 활용
- 협업 신호를 효과적으로 학습

### ✅ 장점 / ❌ 단점

### 추천 모델 선택

| 상황 | 추천 모델 | 이유 |

| **포트폴리오** | EASE | 빠르고 성능 우수, 설명 가능 |

| **리소스 제한** | EASE | 메모리/계산 최소 |

### 데이터 필터링 전략

**EASE 모델의 K30 필터링 과정:**
```
원본 데이터 (31.5M 상호작용)
    ↓
K-core (k=10) 필터링
    ↓
User Activity Range (20-500)
    ↓
Item Popularity Range (20-1000)
    ↓
K-core (k=30) 반복 필터링 (5회)
    ↓
최종 데이터 (1.9M 상호작용)
    ↓
메모리: 41.68GB → 1.42GB (97% 감소)
```

**필터링 의미:**
- **K-core (k=30)**: 모든 노드가 최소 30개 연결 보유 (고립 노드 제거)
- **User Activity (20-500)**: 활동적이지만 합리적 범위의 사용자만
- **Item Popularity (20-1000)**: 중간 정도 인기도의 게임만

## 💡 핵심 인사이트

### EASE가 높은 성능을 보이는 이유

1. **데이터 품질**: K30 필터링으로 매우 정제된 데이터 사용
2. **선형성**: 협업 신호가 명확하면 선형 모델이 매우 효과적
3. **정규화**: 적절한 정규화로 오버피팅 방지
4. **파라미터 수**: 1개의 파라미터만 학습하여 과적합 최소화

### BPR이 EASE보다 낮은 이유

1. **필터링 없음**: 원본 데이터의 노이즈 포함
2. **임베딩 크기**: 64차원이 제한적일 수 있음
3. **학습**: BPR은 더 많은 데이터를 올바르게 학습 필요

### LightGCN이 느린 이유

1. **그래프 구성**: 31.5M 상호작용의 이분 그래프 생성
2. **컨볼루션**: 매 에포크마다 그래프 컨볼루션 계산
3. **대규모 데이터**: 222K 사용자, 50K 아이템의 대규모 학습
