# 진화 설계 필요 — Evolution Design Needed

> **마커 문서.** 진화 설계 자체가 아니라, **"진화 설계가 필요하다"는 사실**을 기록한다.
> 작성일: 2026-07-09 · 상태: 미착수(needs design)

## 요약
현재 추천 파이프라인(EASE + LightGCN 리트리벌 → DCN v2 랭킹 → XGBoost 스태킹 + LLM 레이어)은
**약 1년 전(부트캠프 시점) 설계를 그대로 유지**하고 있다. 그 이후의 recsys 흐름을 반영하지 못했고,
설계 당시 **레드팀(비판적 검증)을 제대로 수행하지 못했다.**
→ **낡은 recsys 흐름을 최신 recsys로 교체하는 진화 설계가 필요하다.**

## 왜 (근거)
1. **노후화 (~1년):** 코어 추천 알고리즘·학습 파이프라인이 부트캠프 당시 그대로다. reactivation·
   backend-refactoring 트랙은 **부활·운영·구조 정리**였을 뿐, **추천 모델 자체의 현대화는 아니었다**
   (코어 알고리즘 변경 없음).
2. **레드팀 미실천:** 설계 당시 가정·평가·실패모드에 대한 비판적 검증이 충분치 않았다.
   오프라인 평가·베이스라인·가정 검증을 **처음부터 다시** 세울 필요.
3. **최신 recsys 미반영:** 리트리벌·랭킹·LLM 통합·평가 관행이 현행 SOTA 대비 재검토 대상.

## 범위 / 프레이밍
- 이 필요성은 **특정 다운스트림 제품 방향과 무관하게**, recsys 자체의 **노후화와 검증 부족**에서
  비롯된다. (외부 제품 아이디어에서 온 편향이 아님.)
- 이 문서는 **"필요하다"는 마커**일 뿐, 설계·구현이 아니다.

## 다음 (이 문서 범위 밖)
진화 설계 착수 시 별도 트랙(`docs/evolution/`)에서:
(1) 현행 파이프라인 레드팀 → (2) 최신 recsys 후보 조사·벤치마크 → (3) 데이터/평가 재정의 →
(4) 교체 로드맵.
